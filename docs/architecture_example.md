## Аналіз функцій застосунку Duolingo за блоками: Frontend, Backend, Database та API

Для аналізу обрано застосунок Duolingo.

*Duolingo* — це навчальний застосунок для вивчення мов. Користувач проходить уроки, виконує вправи, отримує перевірку відповідей, бачить свій прогрес, заробляє бали досвіду та продовжує серію навчальних днів.

Цей застосунок зручно аналізувати, тому що в ньому добре видно взаємодію основних технічних блоків:

```text
Frontend -> API -> Backend -> Database
```

- **Frontend** — те, що бачить користувач і з чим він взаємодіє.

- **Backend** — внутрішня логіка застосунку.

- **Database** — місце зберігання даних.

- **API** — інтерфейс програмування застосунків, тобто “міст” між frontend і backend.

### Функція 1: Відображення екрана уроку

**Основний блок: Frontend**

Коли користувач відкриває урок у Duolingo, він бачить екран із завданням.  
  
Наприклад:
- речення для перекладу;
- варіанти відповідей;
- поле для введення тексту;
- кнопку “Перевірити”;
- індикатор прогресу уроку;
- іконки, кольори, анімації та підказки.

Це належить до *frontend*, тому що саме frontend відповідає за зовнішній вигляд застосунку та взаємодію користувача з екраном.

#### Розподіл по блоках

| Блок | Опис |
| --- | --- |
| Frontend | Показує екран уроку, завдання, кнопки, варіанти відповідей та індикатор прогресу. |
| API | Передає запит на отримання даних уроку. |
| Backend | Готує потрібні дані для уроку. |
| Database | Зберігає завдання, тексти, відповіді та навчальний контент. |

*Frontend* не створює сам навчальний контент. Він лише показує користувачу ті дані, які отримує від backend через API. Але сам екран уроку, його вигляд, кнопки, кліки й візуальні елементи — це частина frontend.

### Функція 2: Надсилання відповіді користувача на перевірку

**Основний блок: API**

Коли користувач виконує вправу, наприклад вводить переклад або вибирає правильне слово, він натискає кнопку “Перевірити”. Після цього відповідь потрібно передати з frontend на backend.

Ця передача відбувається через *API*.

#### Розподіл по блоках

| Блок | Опис |
| --- | --- |
| Frontend | Збирає відповідь користувача з екрана. |
| API | Передає відповідь від frontend до backend. |
| Backend | Отримує відповідь і готує її до перевірки. |
| Database | Може містити правильну відповідь або дані вправи. |

API — це “міст” між frontend і backend. Frontend не повинен напряму звертатися до database. Він надсилає відповідь через API, а backend уже вирішує, що з цією відповіддю робити.

Приклад:

```text
Користувач натискає "Перевірити"
-> Frontend збирає відповідь
-> API передає відповідь на backend
-> Backend перевіряє відповідь
-> API повертає результат на frontend
```

### Функція 3: Перевірка відповіді користувача

**Основний блок: Backend**

Після того як відповідь користувача потрапляє на *backend*, система має визначити, чи вона правильна.

Наприклад, користувач перекладає речення. Backend може порівняти відповідь із правильним варіантом, врахувати альтернативні правильні відповіді або визначити, чи помилка є суттєвою.

Це належить до backend, тому що перевірка відповіді — це внутрішня логіка застосунку.

#### Розподіл по блоках

| Блок | Опис |
| --- | --- |
| Frontend | Показує вправу і дозволяє користувачу дати відповідь. |
| API | Передає відповідь користувача на backend. |
| Backend | Перевіряє правильність відповіді. |
| Database | Зберігає правильні відповіді, можливі варіанти та дані вправи. |

Frontend може показати результат: “Правильно” або “Неправильно”. Але саме рішення про правильність відповіді має приймати backend.

Backend — це “мозок” цієї функції. Він аналізує відповідь, порівнює її з даними вправи та повертає результат.

### Функція 4: Збереження прогресу користувача

**Основний блок: Database**

Після проходження уроку застосунок має зберегти прогрес користувача. 
  
Наприклад:
- який урок завершено;
- скільки XP отримано;
- які помилки були зроблені;
- чи оновилася серія навчальних днів;
- який рівень курсу відкрито наступним.

Це пов’язано з *database*, тому що ці дані потрібно зберігати й використовувати пізніше.

#### Розподіл по блоках

| Блок | Опис |
| --- | --- |
| Frontend | Показує користувачу результат уроку та прогрес. |
| API | Передає дані про завершення уроку між frontend і backend. |
| Backend | Обчислює новий прогрес, XP, streak та доступні уроки. |
| Database | Зберігає оновлений прогрес користувача. |

Database — це “пам’ять” застосунку. Якщо користувач закриє Duolingo і відкриє його пізніше, застосунок має пам’ятати, які уроки вже пройдені та який прогрес було досягнуто.

Backend виконує обчислення, але саме database зберігає результат цих обчислень.

### Функція 5: База вправ і правильних відповідей

**Основний блок: Database**

Щоб Duolingo міг показувати уроки, він повинен мати навчальний контент. 
  
Наприклад:
- курси
- теми
- уроки
- вправи
- речення
- слова
- варіанти відповідей
- правильні відповіді
- аудіофайли
- рівень складності завдань

Це належить до *database*, тому що ці дані потрібно зберігати, оновлювати та використовувати для різних користувачів.

#### Розподіл по блоках

| Блок | Опис |
| --- | --- |
| Frontend | Показує вправи користувачу на екрані. |
| API | Передає запит на отримання вправ. |
| Backend | Вибирає потрібні вправи для конкретного уроку. |
| Database | Зберігає навчальний контент і правильні відповіді. |

Frontend не зберігає всі уроки самостійно. Він отримує потрібні вправи через API. Backend визначає, які саме завдання показати користувачу, а database зберігає сам навчальний матеріал.

Database у цій функції виконує роль сховища навчального контенту.

### Загальна таблиця

| № | Функція | Основний блок | Чому саме цей блок |
| --- | --- | --- | --- |
| 1 | Відображення екрана уроку | Frontend | Користувач бачить цей екран і взаємодіє з ним. |
| 2 | Надсилання відповіді на перевірку | API | API передає відповідь від frontend до backend. |
| 3 | Перевірка відповіді користувача | Backend | Backend виконує логіку перевірки. |
| 4 | Збереження прогресу користувача | Database | Прогрес потрібно зберігати для майбутніх сесій. |
| 5 | База вправ і правильних відповідей | Database | Навчальний контент і відповіді зберігаються в базі даних. |

### Загальна схема взаємодії

```text
Frontend -> API -> Backend -> Database
```

### Приклад повного процесу

1. Користувач відкриває урок.
2. Frontend показує екран уроку.
3. Frontend через API запитує дані уроку.
4. Backend отримує запит.
5. Backend звертається до database.
6. Database повертає вправи та правильні відповіді.
7. Backend передає дані через API.
8. Frontend показує завдання користувачу.
9. Користувач дає відповідь.
10. Frontend надсилає відповідь через API.
11. Backend перевіряє відповідь.
12. Backend оновлює прогрес.
13. Database зберігає новий прогрес.
14. Frontend показує результат користувачу.

### Висновок

На прикладі Duolingo можна побачити, як реальний застосунок поділяється на технічні блоки.

**Frontend** відповідає за інтерфейс: екрани уроків, кнопки, відповіді, візуальні елементи.

**API** відповідає за передачу даних між frontend і backend.

**Backend** відповідає за логіку: перевірку відповідей, розрахунок прогресу, вибір наступних дій.

**Database** відповідає за зберігання даних: профілю користувача, прогресу, уроків, вправ і правильних відповідей.
